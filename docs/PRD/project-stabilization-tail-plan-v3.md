# 武侠人生项目稳定化收尾计划（V3）

## 1. 背景
当前项目已完成基础止血：
- typecheck 通过
- build 通过
- 真实测试门禁（AllTests + IntegrationTests + testGameSimulation）可执行并通过

但最新验证仍暴露收尾缺口：
- 兼容性测试存在“待实现”空洞
- 运行日志出现 Unknown stat: charisma
- 测试环境存在 localstorage 参数警告
- 前端主包体积接近 500KB，存在持续膨胀风险

## 2. 目标
将项目从“可运行”提升到“可持续交付”，完成稳定化最后一公里。

## 3. 范围

### 3.1 In Scope
- 补齐兼容性测试缺口
- 修复属性字典不一致（Unknown stat）
- 清理测试环境配置警告
- 制定并落地首轮包体控制措施

### 3.2 Out of Scope
- 新玩法开发
- 剧情扩展
- UI/美术重设计

## 4. 分阶段计划

### P0-1：兼容性测试补齐
- 将“旧格式兼容性测试待实现”转为可执行断言
- 明确旧格式输入样本与预期输出
- 纳入默认测试门禁

验收：
- AllTests 中不再出现兼容性“待实现”提示
- 兼容路径有通过/失败可判定结果

### P0-2：属性字典一致性修复
- 排查 charisma 等属性在类型定义、数据配置、执行器中的映射
- 统一属性白名单与校验逻辑
- 避免运行时 Unknown stat 日志

验收：
- 真实测试门禁运行中不再出现 Unknown stat: charisma
- 相关属性写入路径有测试覆盖

### P0-3：测试环境警告清理
- 定位并修复 localstorage 参数路径来源
- 区分本地与 CI 的环境初始化策略
- 保证测试运行日志无环境噪音警告

验收：
- 测试全流程中不再出现 localstorage 路径警告

### P1-1：包体控制第一阶段
- 分析入口依赖与可拆分模块
- 优先处理高收益分包点（低风险、可逆）
- 建立包体预算基线并输出报告

验收：
- 主包体积较当前基线（约 486KB）有可量化下降
- 保留 build 通过与功能行为一致性

### P1-2：门禁与文档收口
- 更新测试/发布文档，固化质量门禁与日志标准
- 约束“不得带已知警告上线”

验收：
- 文档中明确发布前检查项：typecheck + build + real test gate + 关键日志零告警

## 5. 风险与缓解
- 风险 1：兼容性补测可能暴露更多历史数据分支
  - 缓解：先定义最小兼容矩阵，逐步扩展
- 风险 2：属性白名单收敛引发旧事件配置失败
  - 缓解：增加配置校验与迁移提示
- 风险 3：包体优化引入加载行为变化
  - 缓解：每次拆包后回归主流程并保留回退点

## 6. 工时评估
- P0-1：0.5-1 人日
- P0-2：0.5-1 人日
- P0-3：0.5 人日
- P1-1：1-1.5 人日
- P1-2：0.5 人日

总计：3-4.5 人日（单人）

## 7. 交付标准
满足以下条件判定收尾完成：
1. npm run typecheck 通过
2. npm run build 通过
3. npm test（真实门禁）通过
4. 不再出现 Unknown stat / localstorage 路径警告
5. 兼容性测试项从“待实现”变为可断言
6. 包体优化结果有量化记录并完成文档更新
