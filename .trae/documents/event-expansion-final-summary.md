## 技术实现亮点

### 1. 统一数据结构
所有事件遵循标准化的 JSON 结构：
```json
{
  "id": "唯一标识符",
  "version": "版本号",
  "category": "事件类别",
  "priority": "优先级 (1-100)",
  "weight": "权重 (1-100)",
  "ageRange": {"min": 最小年龄，"max": 最大年龄},
  "triggers": ["触发器数组"],
  "conditions": ["条件数组"],
  "content": {
    "title": "事件标题",
    "text": "事件描述",
    "description": "简短描述"
  },
  "eventType": "choice/auto/ending",
  "choices": ["选择项（仅 choice 类型）"],
  "autoEffects": ["自动效果（仅 auto/ending 类型）"],
  "requirements": {
    "statRequirements": ["属性需求"],
    "flagRequirements": ["标记需求"]
  }
}
```

### 2. 触发机制多样化
- **age_reach**: 年龄触发（主要机制）
- **random**: 概率触发（增加随机性）
- **flag_based**: 标记触发（事件链依赖）

### 3. 条件系统复杂化
- **属性条件**: `chivalry >= 20`
- **标记条件**: `flags.buddhist_fate == true`
- **复合条件**: `martialPower >= 90 && chivalry >= 60`
- **否定条件**: `!flags.demonic_new_master == true`

### 4. 效果系统丰富
- **stat_modify**: 属性修改（11 种属性）
- **flag_set**: 设置标记（状态追踪）
- **flag_unset**: 移除标记（状态清除）

### 5. 路线分支设计
- **多入口**: 不同条件触发不同路线
- **多分支**: 关键选择点产生分支
- **多结局**: 根据路线和属性决定结局
- **可转换**: 部分路线可相互转换（如邪道赎罪）

## 剧情连贯性保障

### 事件链设计原则
1. **年龄递进**: 事件按年龄顺序触发
2. **条件依赖**: 后续事件依赖前置事件标记
3. **逻辑自洽**: 选择与结果逻辑一致
4. **因果报应**: 侠义值影响结局走向

### 路线转换机制
- **佛门→正道**: 俗家弟子可加入正道
- **正道→邪道**: 侠义值下降可堕入魔道
- **邪道→正道**: 完成赎罪任务可回归
- **医生→中立**: 可选择任何阵营
