### ⚠️ 需要优化的部分

#### 1. 前言事件链未触发

**问题**：新创建的 `prologue.json` 事件链（0-10 岁）没有触发

**原因分析**：
- 0 岁时间段已被 `birth_wuxia_family` 事件占用
- 1-10 岁时间段已被教程事件占用
- 每年事件上限为 5 个，已达到上限

**建议解决方案**：
```
方案 A：将前言事件与现有事件整合
  - 保留出生事件作为前言第一幕
  - 将家族试炼（3 岁）替换部分教程事件
  - 确保事件按叙事顺序触发

方案 B：调整事件触发机制
  - 为前言事件设置更高的优先级
  - 增加"前言事件不受年度上限限制"的机制
  - 确保前言事件在其他事件之前触发

方案 C：移除或合并重复事件
  - 移除 general.json 中的出生事件
  - 用 prologue.json 的事件替代
```

#### 2. 事件 ID 缺失问题

测试检测到大量事件选择缺少 ID，包括：
- career 系列事件：约 20 个选择缺少 ID
- family 系列事件：约 15 个选择缺少 ID  
- elderly 系列事件：约 20 个选择缺少 ID
- hero 系列事件：部分选择缺少 ID

**影响**：
- 无法准确追踪玩家选择
- 影响成就系统和统计功能
- 降低代码可维护性

**建议**：继续补充缺失的事件 ID

#### 3. 自动事件缺少 autoEffects

检测到的问题：
- `family_grandchild_born` 缺少 autoEffects
- `family_reunion` 缺少 autoEffects
- `family_family_honor` 缺少 autoEffects
- `elderly_legend_continues` 缺少 autoEffects

**影响**：自动事件触发时没有任何效果

#### 4. 事件重复触发

观察到以下事件多次触发：
- `魔教长老`：触发了 7 次以上（43-59 岁期间反复触发）
- `家族团圆`：触发了 5 次以上
- `江湖传说`：触发了 5 次以上（75-79 岁连续触发）
- `富可敌国`：触发了 3 次

**建议**：
- 为重复事件添加冷却时间
- 添加 flag 防止同一事件多次触发
- 增加事件变体，丰富内容
