# 爱情 - 婚姻冲突系统设计文档

## 1. 问题分析

### 当前系统缺陷
1. **爱情线与婚姻线割裂**
   - 爱情线：与明月等恋人的浪漫故事，但不影响婚姻
   - 婚姻线：简单的"已婚"flag 设置，缺乏情感基础
   - 两者完全独立，没有交集和冲突

2. **婚姻关系表现形式单一**
   - 只有"已婚/未婚"两种状态
   - 缺乏婚姻质量、夫妻感情深度等维度
   - 没有配偶角色的人格化塑造

3. **缺乏戏剧冲突**
   - 没有婚外情、情感纠葛等现实冲突
   - 家庭责任与个人情感追求无矛盾
   - 婚姻选择缺乏长远影响

## 2. 系统设计目标

### 2.1 核心设计理念
- **关联性**: 爱情线的发展直接影响婚姻质量和配偶选择
- **冲突性**: 婚姻责任与个人情感追求产生矛盾
- **动态性**: 婚姻关系随时间、事件动态变化
- **后果性**: 情感选择产生长期影响和后果

### 2.2 新增系统特性
1. **配偶系统**: 将恋人转化为配偶，有独立人格和关系值
2. **婚姻质量**: 引入"婚姻满意度"指标 (0-100)
3. **情感冲突**: 设计婚外情、旧爱重逢等冲突事件
4. **家庭责任**: 子女教育、赡养老人等责任事件
5. **社会压力**: 家族期望、社会舆论等外部压力

## 3. 系统架构设计

### 3.1 数据结构扩展

```typescript
interface MarriageState {
  isMarried: boolean;
  spouseId: string;           // 配偶 ID
  spouseName: string;         // 配偶姓名
  marriageType: 'arranged' |  'love' | 'pragmatic'; // 婚姻类型
  marriageQuality: number;    // 婚姻质量 (0-100)
  spouseAffection: number;    // 配偶好感度 (0-100)
  marriedAt: number;          // 结婚年龄
  hasChildren: boolean;
  children: ChildState[];
  
  // 情感状态
  extramaritalAffairs: ExtramaritalAffair[];  // 婚外情记录
  pastLovers: string[];       // 过往恋人列表
}

interface ExtramaritalAffair {
  loverId: string;
  loverName: string;
  startedAt: number;
  discovered: boolean;        // 是否被发现
  intensity: number;          // 感情深度 (0-100)
}
```

### 3.2 关系网络

```
初恋情人 (明月) ──┬──> 结婚 ──> 配偶
                  ├──> 分手 ──> 前女友 ──> 重逢事件
                  └──> 婚外情 ──> 情感冲突

包办婚姻 ────────┬──> 先婚后爱 ──> 美满婚姻
                  └──> 感情淡漠 ──> 婚外情风险↑
```

## 4. 冲突事件设计

### 4.1 婚前冲突 (爱情→婚姻转化)

#### 事件 1: 爱情与现实的抉择
- **触发条件**: 与明月好感度≥60，年龄≥22
- **冲突点**: 
  - 明月出身贫寒，家族反对
  - 门当户对的千金提亲
- **选择**:
  1. 坚持真爱 (获得明月，家族关系↓，财富↓)
  2. 接受联姻 (获得富家千金，明月线结束，侠义↓)
  3. 拖延不决 (明月嫁作他人妇，孤独终身风险)

#### 事件 2: 事业与爱情的平衡
- **触发条件**: 明月线进行中，魔教/正派路线选择
- **冲突点**: 
  - 加入魔教会被正道不齿，明月家人反对
  - 加入正派可能牺牲个人发展
- **选择**:
  1. 为明月加入正派 (明月线继续，个人发展受限)
  2. 坚持魔道路线 (明月分手，获得魔道支持)
  3. 隐瞒身份 (双重生活，暴露风险)

### 4.2 婚后冲突 (婚姻维护)

#### 事件 3: 旧爱重逢
- **触发条件**: 已婚，明月未结婚或守寡
- **冲突点**: 
  - 明月出现在生活中，唤起旧情
  - 对现任配偶的愧疚 vs 对旧爱的眷恋
- **选择**:
  1. 断绝往来 (婚姻质量↑，个人心情↓)
  2. 保持友谊 (婚姻质量波动，风险积累)
  3. 发展婚外情 (刺激但高风险)

#### 事件 4: 婚外情暴露危机
- **触发条件**: 有婚外情，且 intensity≥50
- **冲突点**: 
  - 配偶发现蛛丝马迹，开始怀疑
  - 情人要求名分，步步紧逼
- **选择**:
  1. 坦白并结束婚外情 (配偶原谅度检定，可能离婚)
  2. 继续隐瞒 (压力↑，暴露风险↑)
  3. 抛弃家庭 (名誉↓，侠义↓，孤独结局)

#### 事件 5: 家族压力与传宗接代
- **触发条件**: 已婚 3 年无子
- **冲突点**: 
  - 家族催促生子，安排纳妾
  - 配偶反对纳妾，婚姻质量↓
- **选择**:
  1. 坚持不纳妾 (配偶满意度↑，家族关系↓)
  2. 纳妾 (配偶满意度↓↓，家族关系↑)
  3. 过继子嗣 (折中方案)

### 4.3 家庭责任冲突

#### 事件 6: 子女教育分歧
- **触发条件**: 有子女，子女年龄≥10
- **冲突点**: 
  - 你希望子女习武，继承衣钵
  - 配偶希望子女读书，考取功名
- **选择**:
  1. 强制子女习武 (配偶不满，子女可能叛逆)
  2. 尊重子女意愿 (家庭和谐)
  3. 文武兼修 (子女压力↑)

#### 事件 7: 赡养老人 vs 个人发展
- **触发条件**: 父母年龄≥60，你正处事业上升期
- **冲突点**: 
  - 父母需要照顾，希望你回家
  - 事业关键期，无法离开
- **选择**:
  1. 回家照顾父母 (事业机会丧失，孝道↑)
  2. 请人照顾，继续事业 (父母失望，事业↑)
  3. 接父母同住 (家庭负担↑)

## 5. 婚姻质量系统

### 5.1 影响因素

```
婚姻质量 = 基础值 (50) 
         + 配偶好感度 × 0.3 
         + 婚姻类型加成 (爱情婚姻 +20, 包办婚姻 -10)
         - 婚外情惩罚 (每个 -30)
         - 重大冲突事件惩罚
```

### 5.2 婚姻质量等级

| 等级 | 分数范围 | 效果 |
|------|----------|------|
| 美满 | 80-100 | 配偶支持事业，家庭和睦，子女成长良好 |
| 和谐 | 60-79 | 正常婚姻生活，偶有小摩擦 |
| 平淡 | 40-59 | 感情淡漠，但维持表面和平 |
| 危机 | 20-39 | 频繁争吵，可能分居 |
| 破裂 | 0-19 | 离婚风险极高，家庭破碎 |

### 5.3 离婚机制

**离婚触发条件** (满足任一):
- 婚姻质量 ≤ 10 持续 5 年
- 婚外情被发现且配偶原谅度检定失败
- 重大伤害事件 (如家暴、背叛)

**离婚后果**:
- 财产分割 (金钱 -30%~-50%)
- 子女抚养权争夺 (可能失去子女)
- 名誉下降 (声望 -20)
- 侠义值变化 (根据离婚原因)

## 7. 预期效果

### 改进前
- 爱情线：明月等恋人，但没有后续
- 婚姻线：简单设置"已婚"flag
- 两者无关联，缺乏戏剧冲突

### 改进后
- **爱情影响婚姻**: 与明月结婚→婚姻质量高但经济压力大；联姻→财富↑但情感空虚
- **婚姻约束爱情**: 已婚状态下发展新恋情→婚外情风险→家庭危机
- **责任与情感冲突**: 家庭责任 vs 个人发展，旧爱 vs 现任，道德 vs 欲望
- **长期后果**: 每个选择都影响最终人生结局
