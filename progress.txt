## Codebase Patterns

- Blocker log substrings and gate child `NODE_OPTIONS` hygiene live in `tests/qualityGatePolicy.ts`; import this module from auxiliary runners instead of `runRealTestGate.ts` to avoid accidental suite execution on import.
- `ConditionEvaluator` must embed replaced `flags.*` / `player.*` string values using `JSON.stringify` (or equivalent) so compared literals like `'unconventional'` never degrade into bare identifiers inside `new Function`.
- `DifficultyManager` must guard all `localStorage` access with a real browser check (`typeof window !== 'undefined'` + `window.localStorage`); Node's experimental `globalThis.localStorage` triggers `--localstorage-file` warnings during gates.
- Report artifacts are generated under `public/reports/`; keep only `manifest.json` tracked and ignore volatile report files.
- Real test gate should execute suites sequentially with aggregated exit status (do not short-circuit with shell `&&`), so full regression signal is always visible.
- Keep `state.flags` and `state.player.flags` synchronized when mutating flags to avoid read-path inconsistencies across engine and tests.
- Compatibility stories should replace placeholder logs with executable legacy/new-format assertions in `tests/AllTests.ts`.
- For first-stage bundle optimization, prefer low-risk Rollup `manualChunks` splits in `vite.config.ts` (e.g., separate `node_modules` and icon libraries) before touching gameplay code paths.
- For `STAT_MODIFY`, treat recognized player stat keys with `undefined` values as legacy-uninitialized state (default to `0`) and warn only for truly unknown stat keys.
- Integration baseline states (e.g., `GameTestFramework.createTestState`) must include every commonly modified stat field (such as `charisma`, `constitution`, `comprehension`) to avoid `Unknown stat` warnings that mask real regressions.
- Release validation should treat warning hygiene as a first-class gate: pass checks must be accompanied by no unresolved critical warnings in test/build logs.
