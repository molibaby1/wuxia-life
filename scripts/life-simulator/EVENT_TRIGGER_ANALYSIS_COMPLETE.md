## 二、根本原因分析

### 2.1 权重配置不合理（核心原因）

通过详细分析同年龄段的竞争事件，发现权重配置存在严重问题：

| 年龄段 | 日常事件权重 | 特殊事件权重 | 权重比 |
|--------|-------------|-------------|--------|
| 8-12 岁 | 50 (修炼) | 25 (恶棍) | **2:1** |
| 14-24 岁 | 60 (修炼) | 20 (仇家) | **3:1** |
| 16-18 岁 | 60 (修炼) | 35 (意中人) | **1.7:1** |
| 18-35 岁 | 60 (修炼) + 100 (仙草) | 30 (比武) | **5.3:1** |

**触发概率计算**:

```
恶棍欺人:  25 / (25+50+15+10) = 25%  ⚠️ 较低
仇家寻仇:  20 / (20+60+15+20+10) = 16%  ❌ 过低
意中人:    35 / (35+60+25+20) = 25%  ⚠️ 较低
比武大会:  30 / (30+60+100+50) = 12.5%  ❌ 过低
```

### 2.2 年龄范围过窄（加剧因素）

- **意中人事件**: 仅在 16-18 岁（3 年窗口期）
  - 如果这 3 年都被高权重事件覆盖，永久无法触发
  - 实际触发概率 = 1 - (1-0.25)^3 ≈ 58%（理论值）
  - 但考虑到随机扰动，实际可能更低

- **恶棍欺人**: 8-12 岁（5 年窗口期）
  - 理论触发概率 = 1 - (1-0.25)^5 ≈ 76%
  - 但实际测试中经常被日常修炼覆盖

### 2.3 随机扰动的局限性

虽然添加了 0.8-1.2 的随机系数，但权重差距过大时效果有限：

```
示例：仇家寻仇 vs 日常修炼
- 仇家：20 * 1.2 = 24（最大）
- 修炼：60 * 0.8 = 48（最小）
- 结果：48 > 24，修炼仍然优先
```

### 2.4 自动节点跳过机制

自动节点（autoNext=true）触发后直接 age+1，可能导致：
- 跳过特定年龄的事件（如仙草事件只能在 21 岁触发）
- 连续多年触发同一类自动事件（如"刻苦修炼"）

```typescript
private getAvailableNodes(state: PlayerState, allNodes: StoryNode[]): StoryNode[] {
  // 1. 将事件分为特殊事件池和日常事件池
  const specialNodes = allNodes.filter(node => 
    node.id.includes('bully') || 
    node.id.includes('enemy') || 
    node.id.includes('love') || 
    node.id.includes('tournament') ||
    node.id.includes('immortal')
  );
  
  const normalNodes = allNodes.filter(node => 
    node.id.includes('normal') || 
    node.id.includes('age_')
  );
  
  // 2. 检查是否有可用的特殊事件
  const availableSpecial = specialNodes.filter(
    node => node.minAge <= state.age &&
            (node.maxAge === undefined || node.maxAge >= state.age) &&
            (!node.condition || this.evaluateCondition(node.condition, state))
  );
  
  // 3. 如果有特殊事件，50% 概率优先触发特殊事件
  if (availableSpecial.length > 0 && Math.random() < 0.5) {
    return availableSpecial.sort((a, b) => {
      const weightA = (a.weight || 0) * (0.8 + Math.random() * 0.4);
      const weightB = (b.weight || 0) * (0.8 + Math.random() * 0.4);
      return weightB - weightA;
    });
  }
  
  // 4. 否则从日常事件中选择
  const availableNormal = normalNodes.filter(/* 同上 */);
  return availableNormal.sort(/* 同上 */);
}
```

**预期效果**:
- 特殊事件有独立的 50% 触发机会
- 不受日常事件高权重影响
- 触发概率可精确控制

## 七、总结

### 核心问题

**权重配置不合理**是导致四个特定事件无法触发的根本原因。日常修炼事件的权重（60）远高于特殊事件（20-35），导致特殊事件触发概率过低（12.5%-25%）。

### 解决方案

推荐采用**组合方案**：
1. 调整事件权重（立即见效）
2. 扩展年龄范围（增加窗口期）
3. 分类事件池（长期优化）

### 影响评估

- **影响范围**: 仅影响事件触发概率，不影响游戏逻辑
- **风险评估**: 低风险，只调整数值配置
- **回滚方案**: 保留原权重配置，可随时恢复
