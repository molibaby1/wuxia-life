# 事件系统扩展细化计划

## 一、现状分析

### 1.1 现有事件统计
- **总事件数**: 105+ 个
- **文件数量**: 13 个 JSON 文件（8 个有内容，5 个为空）
- **覆盖路线**: 通用主线、爱情、官场、丐帮、边地、魔教
- **缺失路线**: 佛门、正道、邪道（文件存在但为空）

### 1.2 现有系统特点
✅ **优点**:
- 完整的人生阶段覆盖（出生→童年→青年→中年→老年→结局）
- 多路线并行设计
- 丰富的选择分支和结局
- 属性、标记、关系系统完善
- 事件间依赖关系清晰

⚠️ **不足**:
- 部分路线内容为空（佛门、正道、邪道）
- 日常事件较少（仅 training.json 中有少量）
- 随机事件/奇遇事件不足
-  NPC 互动事件较少
- 生活技能相关事件缺失
- 江湖声望系统事件不完善
- 经济系统事件较少
- 战斗/比武事件细节不足

---

## 二、扩展目标

### 2.1 内容扩展目标
1. **填充空文件**: 完成佛门、正道、邪道路线事件
2. **新增路线**: 添加医生、商人、刺客等特殊职业路线
3. **丰富日常**: 增加日常修炼、生活技能事件
4. **奇遇系统**: 添加随机奇遇、秘境探险事件
5. **社交系统**: 增加师徒、结拜、仇敌等关系事件
6. **江湖事件**: 添加江湖传闻、武林浩劫等大事件

### 2.2 数量目标
- 总事件数提升至 **300+ 个**
- 每条主要路线至少 **30+ 个事件**
- 日常事件达到 **50+ 个**
- 奇遇事件达到 **40+ 个**
- 结局数量达到 **20+ 种**

## 四、技术实现方案

### 4.1 数据结构优化
```json
{
  "id": "事件唯一标识",
  "version": "版本号",
  "category": "事件类别",
  "priority": "优先级",
  "weight": "权重",
  "ageRange": {"min": 最小年龄，"max": 最大年龄},
  "triggers": [{"type": "触发类型", "value": 触发值}],
  "conditions": [{"type": "条件类型", "expression": "条件表达式"}],
  "content": {
    "text": "事件描述文本",
    "title": "事件标题",
    "description": "简短描述"
  },
  "eventType": "事件类型 (choice/auto/ending)",
  "choices": [/* 选择项数组 */],
  "autoEffects": [/* 自动效果数组 */],
  "requirements": {
    "statRequirements": [/* 属性需求 */],
    "flagRequirements": [/* 标记需求 */]
  },
  "metadata": {
    "createdAt": 创建时间戳，
    "updatedAt": 更新时间戳，
    "author": "作者",
    "tags": ["标签数组"],
    "enabled": true
  }
}
```

### 4.2 新增系统支持

1. **声望系统**: 添加 reputation 属性，0-1000 分
2. **医术系统**: 添加 medicalSkill 属性
3. **商业系统**: 添加 businessSkill 属性
4. **暗杀系统**: 添加 assassinationSkill 属性
5. **san 值系统**: 添加 sanity 属性（邪道专用）
6. **气运系统**: 添加 luck 属性（奇遇专用）

## 六、质量保证

### 6.1 事件平衡性
- 每个选择项的权重合理分配
- 属性增减数值平衡
- 不同路线强度均衡
- 结局达成难度梯度

### 6.2 剧情连贯性
- 事件间逻辑关系清晰
- 前后事件不冲突
- 路线转换自然流畅
- 结局与过程匹配

### 6.3 文本质量
- 描述文本符合武侠风格
- 对话内容生动有趣
- 选项表达清晰明确
- 无错别字和语病

### 6.4 测试验证
- 单元测试：每个事件独立测试
- 集成测试：事件链连贯测试
- 平衡测试：数值合理性验证
- 用户测试：实际游戏体验

### 7.1 技术风险
- **风险**: JSON 文件过大影响加载速度
- **应对**: 采用分文件存储，按需加载

### 7.2 内容风险
- **风险**: 事件过多导致剧情混乱
- **应对**: 严格控制触发条件，确保逻辑清晰

### 7.3 平衡风险
- **风险**: 某些路线过强或过弱
- **应对**: 多次测试调整，收集反馈优化

### 7.4 进度风险
- **风险**: 开发周期过长
- **应对**: 分阶段实施，优先完成核心内容

## 总结

本计划旨在将现有的 105+ 个事件扩展至 300+ 个事件，通过五个阶段的实施：
1. 填充空文件路线（80 个事件）
2. 新增特殊职业路线（60 个事件）
3. 丰富日常和奇遇系统（80 个事件）
4. 完善系统机制（30 个事件）
5. 添加大型剧情事件（50 个事件）

最终实现一个更加丰富、完整、平衡的武侠人生事件系统，为玩家提供多样化的游戏体验和更高的可玩性。
