# 爱情事件逻辑修复总结

## 问题描述

用户报告了一个严重的逻辑矛盾：
- 玩家已经**迎娶明月**成为妻子
- 但后续仍然触发"暗中相助"、"旧爱重逢"等事件
- 这些事件把明月当作前女友，而不是妻子

## 根本原因

爱情事件的条件检查不够严谨，**没有正确区分"明月是妻子"和"明月是前女友"两种状态**。

具体问题分析：

### 问题 1: "暗中相助"事件
**原条件**: `flags.has("love_misunderstood") && !flags.has("love_secret_help")`

**问题**: 
- 只检查是否有误解，不检查明月是否已嫁给别人
- 玩家娶了明月为妻，或者明月嫁给别人，都会触发"暗恋者暗中相助"的剧情
- 逻辑上不合理：夫妻之间不需要"暗中相助"

### 问题 2: "生死相救"事件  
**原条件**: `flags.has("love_secret_help") && !flags.has("love_life_or_death")`

**问题**:
- 只检查前置事件，不检查明月是否是妻子
- 已婚状态下还会触发"与明月共陷险境"的恋爱剧情

### 问题 3: "旧爱重逢"事件
**原条件**: `flags.married == true && flags.mingyue_single == true && !flags.mingyue_fiancee`

**问题**:
- 检查了玩家已婚、明月单身
- 但**没有检查玩家的妻子是否是明月**
- 如果玩家娶了明月，明月就不是"旧爱"，而是"妻子"

## 修复方案

### 修复 1: "暗中相助"事件

**新条件**: 
```
flags.has("love_misunderstood") && 
!flags.has("love_secret_help") && 
!flags.spouse_mingyue && 
!flags.mingyue_married_other
```

**逻辑**:
- `!flags.spouse_mingyue`: 明月不是玩家的妻子
- `!flags.mingyue_married_other`: 明月没有嫁给别人
- 只有明月是玩家的前女友（未嫁）时，才会触发"暗中相助"

### 修复 2: "生死相救"事件

**新条件**:
```
flags.has("love_secret_help") && 
!flags.has("love_life_or_death") && 
!flags.spouse_mingyue
```

**逻辑**:
- `!flags.spouse_mingyue`: 明月不是玩家的妻子
- 只有明月是前女友时，才会触发"生死相救"的恋爱剧情

### 修复 3: "旧爱重逢"事件

**新条件**:
```
flags.married == true && 
flags.mingyue_single == true && 
!flags.mingyue_fiancee && 
!flags.spouse_mingyue
```

**逻辑**:
- `!flags.spouse_mingyue`: 明月不是玩家的妻子
- 只有明月是前女友时，才会触发"旧爱重逢"

## 修复后的逻辑流程

### 场景 1: 玩家迎娶明月
```
选择"迎娶明月" 
  → 设置 flags.spouse_mingyue = true
  → "暗中相助" ❌ 不触发（明月是妻子）
  → "生死相救" ❌ 不触发（明月是妻子）
  → "旧爱重逢" ❌ 不触发（明月是妻子）
  → 触发夫妻相关事件 ✅
```

### 场景 2: 玩家接受联姻，明月嫁给别人
```
选择"接受联姻"
  → 设置 flags.mingyue_married_other = true
  → "暗中相助" ❌ 不触发（明月已嫁人）
  → "生死相救" ❌ 不触发（明月已嫁人）
  → "旧爱重逢" ❌ 不触发（明月已嫁人）
  → 可能触发"旧爱重逢"的变体事件（如果明月离婚/守寡）
```

### 场景 3: 玩家与明月相恋但未结婚
```
恋爱但未结婚
  → "暗中相助" ✅ 触发（明月是女友）
  → "生死相救" ✅ 触发（明月是女友）
  → "爱情与现实的抉择" ✅ 触发
  → 根据选择走向不同结局
```

## 技术实现

### 修改的文件
1. **love.json**
   - `love_secret_help`: 添加配偶和明月婚姻状态检查
   - `love_life_or_death`: 添加配偶检查

2. **love-marriage-conflict.json**
   - `marriage_old_lover_reunion`: 添加配偶检查

### 关键 Flags 说明
- `spouse_mingyue`: 明月是玩家的妻子
- `mingyue_married_other`: 明月嫁给别人
- `mingyue_fiancee`: 明月是玩家的未婚妻
- `mingyue_single`: 明月单身

## 修复效果

### 修复前
- ❌ 已婚玩家还会触发"暗中相助"（暗恋者行为）
- ❌ 明月是妻子还会触发"旧爱重逢"（前女友剧情）
- ❌ 逻辑混乱，玩家体验极差

### 修复后
- ✅ 已婚玩家不再触发前女友剧情
- ✅ 明月嫁给别人后不再触发恋爱剧情
- ✅ 只有明月是前女友时才会触发相应事件
- ✅ 逻辑清晰，符合现实情理

## 后续优化建议

1. **添加夫妻专属事件**
   - 如果明月是妻子，触发"夫妻同心"、"明月支持"等正面事件
   - 而不是"暗中相助"这种暗恋者行为

2. **完善明月状态机**
   ```
   恋人 → 未婚妻 → 妻子
        ↓
      前女友 → 嫁作他人妇
              ↓
            守寡/离婚 → 可重逢
   ```

3. **添加状态转换事件**
   - 明月出嫁事件（如果玩家未娶）
   - 明月守寡/离婚事件（触发重逢剧情的前置）

4. **扩展配偶系统**
   - 为不同配偶（明月、富家千金等）添加独立人格
   - 配偶之间的互动事件
   - 配偶对玩家事业的支持/阻碍

## 总结

本次修复解决了爱情事件中的严重逻辑矛盾，确保：
- **明月是妻子** → 触发夫妻事件，不触发前女友剧情
- **明月已嫁人** → 不触发恋爱剧情
- **明月是前女友** → 触发相应的恋爱/重逢剧情

现在爱情 - 婚姻系统的事件触发逻辑更加严谨和合理，大大提升了玩家的沉浸感和游戏体验。
