## 系统设计特色

### 1. 多路线网状结构
```
出身 → 主线 → 职业路线（8 条）
         ↓
      日常/奇遇
         ↓
      系统机制（声望/经济/人际）
         ↓
      大型剧情（4 条史诗线）
         ↓
      结局（20+ 种）
```

### 2. 属性系统整合
- **核心属性**: 武力、内力、外功、轻功、体质、悟性
- **衍生属性**: 侠义、声望、金钱、魅力
- **特殊属性**: 医术、商业、暗杀、气运

### 3. 标记（Flag）系统
- **路线标记**: `buddhist_monk`, `orthodox_member`, `demonic_corrupted`
- **进度标记**: `meeting_champion`, `war_hero`, `crisis_savior`
- **关系标记**: `has_master`, `has_sworn_siblings`, `has_life_debt`
- **成就标记**: `reputation_legend`, `war_legend`, `crisis_legend`, `dynasty_emperor`

### 4. 动态事件链
- **前置条件**: 年龄、属性、标记、随机概率
- **分支选择**: 多选项、多结果、多影响
- **后续影响**: 改变属性、设置标记、解锁事件

### 5. 道德选择系统
- **侠义考验**: 救人/旁观、正义/自保
- **阵营选择**: 正道/魔教/中立、朝廷/起义
- **利益权衡**: 金钱/侠义、权力/自由、生命/名誉

## 技术亮点

### 1. 事件优先级系统
- **priority**: 决定事件触发顺序（1-100）
- **weight**: 决定事件触发概率（1-100）
- **ageRange**: 限制事件触发的年龄范围

### 2. 条件表达式系统
- **属性条件**: `martialPower >= 100`
- **标记条件**: `flags.war_hero == true`
- **复合条件**: `chivalry >= 50 && reputation >= 200`

### 3. 效果系统
- **stat_modify**: 修改属性值
- **flag_set**: 设置标记
- **flag_unset**: 清除标记

### 4. 结局系统
- **endingType**: 结局类型标识
- **endingTitle**: 结局标题
- **endingDescription**: 结局描述文本
