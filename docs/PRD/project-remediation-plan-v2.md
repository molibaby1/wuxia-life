# 武侠人生项目修复计划（V2）

## 1. 背景与问题证据
基于最新执行证据（2026-04-24）：
- tests/AllTests.ts 存在核心失败：Flag 设置效果断言失败（15/16）
- tests/IntegrationTests.ts 直接失败：ERR_MODULE_NOT_FOUND，缺失 src/data/lines/demonic.json
- tests/testGameSimulation.ts 持续报警：教程事件未触发、学识与战斗成长偏低

结论：当前项目虽可构建，但真实回归链路不通过，属于可出包但不可保证行为正确的高风险状态。

## 2. 目标
在不新增玩法功能前提下，恢复项目的真实可验证性与最小交付可信度。

### 2.1 总目标
- 让默认验证链路能够真实发现核心回归
- 清除当前三条已知红灯（Flag 失败、资源断链、教程触发异常）
- 建立可执行的 PR 质量门禁基础

## 3. 范围

### 3.1 In Scope
- 测试入口与校验链路修复
- AllTests 核心失败修复（Flag 设置）
- IntegrationTests 资源引用/文件一致性修复
- 教程触发与成长权重基础修正
- PR 验证命令与文档收口

### 3.2 Out of Scope
- 新剧情扩展
- 新系统开发
- 美术、UI 体验优化
- 大规模玩法重构

## 4. 实施计划（先后顺序固定）

### P0-1：测试门禁改为真实链路
- 修改默认 npm test 的执行目标，使其覆盖：
  - tests/AllTests.ts
  - tests/IntegrationTests.ts
  - tests/testGameSimulation.ts
- 保持 validate 依赖 test，禁止假绿

验收：
- npm test 必须执行上述三组测试，而不是仅导出类型测试

### P0-2：修复 AllTests 的 Flag 失败
- 定位 Flag 设置效果失败根因（Effect 执行器/状态写入/断言口径）
- 修复逻辑，保证 flag 状态按预期落库

验收：
- tests/AllTests.ts 全通过

### P0-3：修复 IntegrationTests 断链
- 处理 demonic.json 缺失问题：
  - 方案 A：恢复文件并纳入资源管理
  - 方案 B：修正引用到现有替代文件并更新测试
- 同步检查同目录其他潜在悬空引用

验收：
- tests/IntegrationTests.ts 可完整运行并通过

### P1-1：修复教程事件触发异常
- 排查教程事件年龄范围、触发条件、优先级与互斥关系
- 保证模拟流程中教程事件可按预期出现

验收：
- tests/testGameSimulation.ts 不再出现教程事件未触发

### P1-2：修复成长曲线偏离
- 调整学习/修炼相关事件权重与分布
- 避免关键属性长期贴地增长

验收：
- tests/testGameSimulation.ts 不再持续给出学识/战斗成长过低核心警告

### P1-3：质量门禁文档化
- 更新 README/测试说明，明确合并前强制命令
- 统一本地与 CI 验证口径

验收：
- 文档中明确质量门禁：typecheck + build + test(真实链路)

## 5. 风险与缓解
- 风险 1：修复资源断链时引入更多历史引用问题
  - 缓解：同目录进行一次引用扫描，避免仅修单点
- 风险 2：修教程触发导致其他事件权重被挤压
  - 缓解：每次权重调整后固定跑模拟测试并记录对比
- 风险 3：默认测试变重导致开发者绕过
  - 缓解：增加分层命令（快速/完整）但 PR 只认完整链路

## 7. 交付标准
满足以下条件方可认定修复完成：
1. npm run typecheck 通过
2. npm run build 通过
3. npm test（真实链路）通过
4. 三条当前红灯问题全部关闭
5. 文档更新并可用于团队执行
