## ⚠️ 发现的问题

### 问题 1: 20 岁事件过于密集（严重）

**现象**: 20 岁一年内触发了 **13 个事件**

```
1. 并肩同行 (auto)
2. 家族阻碍 (choice)
3. 情敌出现 (choice)
4. 别离 (auto)
5. 重逢 (choice)
6. 误会 (choice)
7. 暗中相助 (auto)
8. 生死相救 (choice)
9. 辞别师门 (choice)
10. 边地奇缘 (choice)
11. 互市通道 (choice)
12. 边地试炼 (choice)
13. 边地巡行 (auto)
```

**问题分析**:
- ❌ **不符合常理**: 一年内经历 13 个重大事件，平均每月超过 1 个
- ❌ **节奏过快**: 爱情线（7 个事件）+ 边地线（4 个事件）+ 师门线（1 个事件）+ 仕途线（1 个事件）同时爆发
- ❌ **体验差**: 玩家没有喘息和消化的时间

**合理设计**:
- ✅ 每年触发 **1-3 个重大事件** 为宜
- ✅ 重要剧情线应该**分散在不同年龄**
- ✅ 同一剧情线的事件应该有**时间间隔**

**根本原因**:
1. 事件触发条件过于宽松（年龄范围重叠）
2. 缺少事件间隔控制机制
3. 多条剧情线同时满足触发条件
4. 没有"冷却时间"概念

---

### 问题 2: 事件触发不均衡

**统计数据**:

| 年龄段 | 事件数 | 评价 |
|--------|--------|------|
| 0-10 岁 | 11 个 | ✅ 合理 |
| 11-19 岁 | 10 个 | ✅ 合理 |
| **20 岁** | **13 个** | ❌ **过多** |
| 21-25 岁 | 8 个 | ✅ 合理 |
| 26-34 岁 | 9 个 | ⚠️ 偏少 |
| 35-54 岁 | 20 个 | ⚠️ 平均但偏少 |
| 55-80 岁 | 26 个 | ✅ 合理 |

**问题**:
- 20 岁事件密度是其他年龄段的 **5-8 倍**
- 26-34 岁正值青年，事件反而稀少
- 中年期（35-54 岁）缺乏高潮事件

---

### 问题 3: 多条剧情线冲突

**20 岁同时触发的剧情线**:
- 💕 爱情线：7 个事件（家族阻碍、情敌、别离、重逢、误会、暗中相助、生死相救）
- 🏜️ 边地线：4 个事件（边地奇缘、互市通道、边地试炼、边地巡行）
- 🏔️ 师门线：1 个事件（辞别师门）
- 🏛️ 仕途线：1 个事件（入仕机会，21 岁触发）

**不合理之处**:
- 主角不可能同时：
  - 经历完整的爱情长跑（相遇→阻碍→情敌→别离→重逢→误会→相救）
  - 前往边地历练（随行边地→护商保路→骑射猎场→边地巡行）
  - 辞别师门下山
  - 准备科举入仕

**时间冲突**:
- 爱情线 7 个事件至少需要 **3-5 年** 才合理
- 边地线 4 个事件至少需要 **2-3 年** 才合理
- 现在全部压缩在 1 年内完成

---

### 问题 4: 缺少日常事件填充

**统计数据**:
- 总记录 107 年，90 个事件触发
- 但 **26-34 岁**（9 年）只有 9 个事件
- 平均每年不到 1 个事件

**问题**:
- 没有事件的年份显示"⚠️ 无可用事件"
- 缺少日常修炼、生活琐事等小事件填充
- 导致"要么闲得发慌，要么忙得爆炸"

## 🔍 技术原因分析

### 1. 触发条件设计问题

**现状**:
```json
{
  "id": "love_family_obstacle",
  "ageRange": {"min": 20, "max": 25},
  "triggers": [
    {"type": "age_reach", "value": 20}
  ]
}
```

**问题**:
- 年龄范围太宽（20-25 岁，共 6 年）
- 触发条件只有年龄，没有其他限制
- 多个事件都设定在同一个年龄段

**改进建议**:
```json
{
  "id": "love_family_obstacle",
  "ageRange": {"min": 20, "max": 22},
  "triggers": [
    {"type": "age_reach", "value": 20},
    {"type": "expression", "expression": "flags.love_started == true"}
  ],
  "cooldown": 2  // 新增：冷却时间 2 年
}
```

---

### 2. 缺少事件优先级和互斥机制

**现状**:
- 所有事件权重相似
- 没有优先级区分
- 没有互斥关系

**改进建议**:
```json
{
  "id": "love_family_obstacle",
  "priority": 80,  // 高优先级
  "mutuallyExclusive": ["border_encounter", "official_entry"],  // 互斥事件
  "requiresFlag": "love_first_meet"  // 前置条件
}
```

---

### 3. 缺少年度事件数量限制

**现状**:
- 游戏引擎每年可能触发多个事件
- 没有数量上限

**改进建议**:
```typescript
// 在 GameEngineIntegration.ts 中
private maxEventsPerYear = 3;  // 每年最多 3 个重大事件

public selectEvent(): EventDefinition | null {
  const eventsThisYear = this.getEventsThisYear();
  if (eventsThisYear.length >= this.maxEventsPerYear) {
    return null;  // 今年事件已满，不再触发
  }
  // ... 原有逻辑
}
```

## 💡 改进建议

### 短期方案（立即实施）

1. **调整事件年龄分布**
   - 将 20 岁的事件分散到 20-25 岁
   - 爱情线：20 岁 (2 个) → 21 岁 (2 个) → 22 岁 (2 个) → 23 岁 (1 个)
   - 边地线：21 岁 (2 个) → 22 岁 (2 个)

2. **添加事件前置条件**
   - 爱情线事件需要 `flags.love_started == true`
   - 边地线事件需要 `flags.left_sect == true`
   - 仕途线事件需要 `flags.sect_master_approved == true`

3. **实现冷却时间机制**
   - 同一条剧情线的事件至少间隔 1-2 年
   - 重大事件（如生死相救）后至少休息 1 年

---

### 中期方案（1-2 周）

1. **添加日常事件池**
   - 创建 `daily-events.json`（50+ 个小事件）
   - 在没有重大事件的年份触发日常事件
   - 日常事件包括：修炼、采药、打猎、交友等

2. **实现事件优先级系统**
   - 主线剧情：priority 90-100
   - 支线剧情：priority 70-89
   - 日常事件：priority 50-69
   - 随机事件：priority 30-49

3. **添加事件互斥关系**
   - 爱情线和仕途线互斥（不能同时发展）
   - 边地线和师门线互斥（地理位置冲突）

---

### 长期方案（1-2 月）

1. **实现动态难度调整**
   - 根据玩家属性动态调整事件触发率
   - 高武力玩家更容易触发战斗事件
   - 高魅力玩家更容易触发社交事件

2. **添加事件链系统**
   - 将相关事件组织成事件链
   - 事件链有明确的开始、发展、高潮、结局
   - 每条事件链跨越 3-5 年

3. **实现智能事件调度**
   - 使用 AI 算法优化事件分布
   - 确保玩家体验曲线平滑
   - 避免事件堆积和空白期

## ✅ 验证标准

修改后需要满足以下标准：

1. **每年事件数**: 1-5 个为宜，不超过 5 个
2. **剧情线分布**: 同一条剧情线事件间隔≥1 年
3. **年龄段均衡**: 各年龄段事件数差异不超过 3 倍
4. **逻辑合理性**: 事件之间有时间上的合理性
5. **玩家体验**: 有张有弛，高潮与平淡交替

## 🎯 总结

**核心问题**: 20 岁事件过于密集（13 个事件/年），严重不符合常理

**根本原因**:
1. 事件触发条件设计不合理
2. 缺少冷却时间和互斥机制
3. 多条剧情线同时触发
4. 没有年度事件数量限制

**优先级**: 🔴 高优先级（严重影响游戏体验）

**建议立即实施**:
1. 调整事件年龄分布（1-2 天）
2. 添加事件前置条件（1-2 天）
3. 实现冷却时间机制（2-3 天）
4. 添加年度事件限制（1 天）
