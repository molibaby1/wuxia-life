# 路径锁定和互斥机制设计方案

## 问题分析

当前游戏存在严重的路径混杂问题：
- 玩家在40岁时同时经历学者路线（著书立说）、商业路线（金融危机）、魔教路线（魔教长老）
- 这些路径本质上互相冲突，不应同时出现在一个人生中
- 仅靠背景权重无法解决根本问题，需要实现路径互斥机制

## 设计目标

1. 实现发展路径的互斥机制，防止冲突路径同时进行
2. 允许玩家在关键时刻做出人生选择，锁定发展方向
3. 保持游戏的灵活性，允许一定程度的路径交叉

## 解决方案

### 1. 路径分类系统

定义主要发展路径：
- **scholar** (学者) - 专注于学术、文学、教育
- **merchant** (商人) - 专注于商业、财富、贸易
- **martial_artist** (武者) - 专注于武学、战斗、江湖
- **official** (官员) - 专注于仕途、政治、权力
- **demon** (魔道) - 专注于邪道、力量、反叛
- **hermit** (隐士) - 专注于修行、宁静、避世

### 2. 路径锁定机制

#### 2.1 路径强度系统
- 每个路径有一个强度值 (0-100)
- 当路径强度超过阈值（如30），开始对冲突路径产生抑制作用
- 当路径强度超过高阈值（如70），强烈抑制冲突路径

#### 2.2 路径互斥规则
```
学者 ←→ 魔道: 强烈互斥 (学术正道与邪道对立)
商人 ←→ 隐士: 中等互斥 (商业追求与避世理念冲突)
武者 ←→ 隐士: 中等互斥 (江湖纷争与宁静修行冲突)
魔道 ←→ 官员: 强烈互斥 (邪道与正统权力对立)
```

### 3. 实现方案

#### 3.1 在GameState中添加路径跟踪
```typescript
interface PathStrengths {
  scholar: number;
  merchant: number;
  martial_artist: number;
  official: number;
  demon: number;
  hermit: number;
}

interface GameState {
  // ... 其他属性
  pathStrengths: PathStrengths;
  dominantPaths: string[]; // 主导路径
}
```

#### 3.2 修改事件选择逻辑
在 `selectEvent` 方法中添加路径互斥检查：
- 计算事件所属路径
- 检查与当前主导路径的兼容性
- 对冲突路径的事件应用惩罚权重

#### 3.3 路径权重调整公式
```
adjustedWeight = baseWeight * compatibilityFactor
```

其中 `compatibilityFactor` 根据路径互斥程度动态计算。

### 4. 事件配置增强

为事件添加路径信息：
```json
{
  "id": "write_book",
  "category": "scholar",
  "pathAffinity": {
    "scholar": 30,
    "hermit": 10
  },
  "pathConflicts": {
    "demon": 20,
    "official": 5
  }
}
```

### 5. 路径锁定事件

设计关键的路径锁定事件，一旦触发就会大幅提升特定路径强度，并对冲突路径施加长期抑制。

### 6. 权重调整策略

- **轻微冲突** (< 20): 0.8 倍权重
- **中等冲突** (20-50): 0.5 倍权重  
- **强烈冲突** (> 50): 0.2 倍权重
- **极度冲突** (> 80): 0.05 倍权重（几乎不可能触发）

### 7. 特殊情况处理

- 允许少量跨路径事件（如生活事件）
- 早期事件路径锁定较松，后期逐渐严格
- 某些事件可作为路径转换点，允许有限的路径变更
