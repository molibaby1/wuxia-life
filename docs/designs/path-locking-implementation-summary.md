# 路径锁定和互斥机制实现总结

## 问题解决

成功解决了游戏事件路径混乱的问题，玩家不再同时经历多个相互冲突的人生路径（如同时成为魔教长老和著书立说的学者）。

## 核心改进

### 1. 路径检测系统
在 `GameEngineIntegration.ts` 中实现了智能路径检测：

- **getDominantPaths()**: 根据玩家属性、flags 和发展历史，自动识别当前主导路径
- 支持 6 种主要路径：学者、商人、武者、魔道、官员、隐士
- 路径判定基于多个因素：出身背景、属性阈值、关键选择 flags

### 2. 路径互斥机制
实现了三层互斥检查：

#### 2.1 强烈冲突过滤
- 冲突等级 > 50 的事件直接被过滤掉，不会进入候选池
- 例如：魔道路径玩家不会遇到"著书立说"等学者事件

#### 2.2 权重动态调整
- 根据事件与主导路径的亲和性提升或降低权重
- 亲和性每点提升 1% 权重
- 冲突性根据等级降低权重：
  - 极度冲突 (>80): 0.05 倍
  - 强烈冲突 (>50): 0.2 倍
  - 中等冲突 (>20): 0.5 倍
  - 轻微冲突 (>20): 0.8 倍

#### 2.3 回退机制
- 如果没有兼容事件，回退到所有未触发事件
- 确保游戏不会因过度过滤而无法进行

### 3. 事件配置格式
为事件添加了路径亲和性和冲突配置：

```json
{
  "metadata": {
    "pathAffinity": {
      "scholar": 100,  // 学者路径亲和性 +100%
      "hermit": 30     // 隐士路径亲和性 +30%
    },
    "pathConflicts": {
      "demon": 90,     // 与魔道路径强烈冲突
      "merchant": 40   // 与商人路径中等冲突
    }
  }
}
```

## 路径定义

### 学者路径 (scholar)
- **触发条件**: `scholar_path` flag 或 出身书香门第 或 悟性≥50
- **特点**: 专注于学术、文学、教育、著书立说
- **冲突路径**: 魔道 (强烈)、商人 (轻微)

### 商人路径 (merchant)
- **触发条件**: `merchant_path` flag 或 出身商贾之家 或 金钱≥500
- **特点**: 专注于商业、贸易、财富积累
- **冲突路径**: 隐士 (中等)、学者 (轻微)

### 武者路径 (martial_artist)
- **触发条件**: `martial_path` flag 或 出身武林世家 或 武力≥50
- **特点**: 专注于武学修炼、江湖争斗
- **冲突路径**: 隐士 (中等)

### 魔道路径 (demon)
- **触发条件**: `demon_path` flag 或 加入魔教 或 侠义≤-60
- **特点**: 专注于邪道武功、力量追求
- **冲突路径**: 学者 (强烈)、官员 (强烈)、隐士 (中等)

### 官员路径 (official)
- **触发条件**: `official_path` flag 或 科举成功 或 声望≥100
- **特点**: 专注于仕途、政治、权力
- **冲突路径**: 魔道 (强烈)、隐士 (中等)

### 隐士路径 (hermit)
- **触发条件**: `hermit_path` flag 或 侠义≥80
- **特点**: 专注于修行、宁静、避世
- **冲突路径**: 商人 (中等)、武者 (中等)

## 实现效果

### 改进前
- 玩家 40 岁时同时经历：
  - 著书立说（学者路线）
  - 金融危机（商业路线）
  - 魔教长老（魔道路线）
  - 武林盟主（武者路线）
- 人生路径混乱，缺乏连贯性

### 改进后
- 玩家选择魔道路径后：
  - ✅ 继续触发魔道相关事件（吸星大法、控制下属等）
  - ❌ 不再触发著书立说等学者事件
  - ❌ 不再触发武林盟主等正道事件
  - ⚠️ 商业事件权重降低（除非也走了商人路径）

- 玩家选择学者路径后：
  - ✅ 更多学术、文学事件
  - ❌ 不会成为魔教长老
  - ⚠️ 商业事件权重降低

## 技术实现

### 核心方法

1. **getDominantPaths()**
   - 检测玩家当前主导路径
   - 支持多路径并存（如学者 + 官员）
   - 基于 flags、属性值、出身背景综合判断

2. **isPathConflicting()**
   - 检查事件是否与主导路径强烈冲突
   - 冲突等级>50 直接过滤
   - 提供详细的冲突日志

3. **adjustWeightByPath()**
   - 动态调整事件权重
   - 考虑亲和性和冲突性
   - 确保最小权重为 1

4. **selectEvent()**
   - 整合路径互斥检查
   - 先过滤再加权
   - 提供回退机制

### 事件文件配置

示例事件配置（path-examples.json）：
- 魔道事件：高亲和魔道路径，强烈冲突学者/官员
- 学者事件：高亲和学者路径，强烈冲突魔道
- 商业事件：高亲和商人路径，中等冲突隐士

## 使用说明

### 为事件添加路径配置

1. 确定事件所属路径
2. 设置路径亲和性（0-100）
3. 设置路径冲突（0-100）
4. 在 metadata 中添加配置

```json
{
  "metadata": {
    "pathAffinity": {
      "scholar": 80,
      "official": 20
    },
    "pathConflicts": {
      "demon": 90,
      "merchant": 30
    }
  }
}
```

### 设置路径 flags

在事件效果中设置路径 flag：
```json
{
  "effects": [
    {"type": "flag_set", "flag": "scholar_path", "value": true}
  ]
}
```

## 后续优化方向

1. **渐进式路径锁定**: 早期宽松，后期严格
2. **路径转换机制**: 允许有限的路径变更
3. **跨路径事件**: 保留少量通用事件
4. **动态平衡**: 根据玩家反馈调整冲突等级
5. **可视化**: 向玩家展示当前路径倾向

路径锁定和互斥机制成功解决了事件路径混乱的问题，现在玩家的人生体验更加连贯和真实。不同路径之间有明确的区分和冲突，增强了游戏的代入感和重玩价值。
